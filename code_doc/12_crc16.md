# CRC16 — Frame Integrity Check

**Files:** `edge/app/crc16.hpp`, `edge/app/crc16.cpp`

---

## What this module is

CRC (Cyclic Redundancy Check) is a mathematical checksum. After transmitting a block of data, you also send a CRC value. The receiver computes the CRC on the received data and compares — if they match, the data almost certainly arrived without errors.

AegisCore uses **CRC-16/CCITT-FALSE**:
- Polynomial: `0x1021`
- Initial value: `0xFFFF`
- No input or output bit-reversal
- XOR-out: `0x0000`

This specific variant is specified in the AC2 IRS (Interface Requirements Specification) §3.1 and used by both the STM32 firmware and the Node.js gateway to validate every AC2 frame.

---

## How CRC-16 works (conceptually)

You can think of CRC as dividing the data (treated as a huge binary number) by a fixed polynomial, and the CRC is the remainder. Both sender and receiver do the same division — if the remainder matches, the data is almost certainly intact.

A corrupted bit or corrupted byte will (with very high probability) produce a different remainder, detecting the error.

---

## Lookup table for speed

```cpp
constexpr std::uint16_t kTable[256] = { ... };
```

Computing CRC from scratch requires 8 XOR and shift operations per byte. The lookup-table approach pre-computes the contribution of all 256 possible byte values — reducing each byte to a single table lookup + XOR. This is 8× faster, important on a 168 MHz MCU processing every incoming byte.

The 256 entries are all pre-computed at compile time using `constexpr`, so the table lives in Flash rather than being computed at boot.

## `CRC16(const std::uint8_t* data, std::uint8_t len)` → `std::uint16_t`

```cpp
std::uint16_t CRC16(const std::uint8_t* data, std::uint8_t len) noexcept
{
    std::uint16_t crc = 0xFFFFU;  // initial value
    for (std::uint8_t i = 0U; i < len; ++i)
    {
        const std::uint8_t tbl_idx =
            static_cast<std::uint8_t>((crc >> 8U) ^ data[i]);
        crc = static_cast<std::uint16_t>((crc << 8U) ^ kTable[tbl_idx]);
    }
    return crc;
}
```

**Line by line:**

`std::uint16_t crc = 0xFFFFU` — initialise the running CRC to `0xFFFF` (CCITT-FALSE specification).

For each byte:

1. `(crc >> 8U)` — take the high byte of the current CRC value
2. `^ data[i]` — XOR it with the current input byte
3. Result is used as a table index (0–255)
4. `(crc << 8U)` — shift the current CRC left by 8 (discard the high byte we just used)
5. `^ kTable[tbl_idx]` — XOR with the pre-computed table entry

After processing all `len` bytes, the 16-bit `crc` is the checksum.

**Parameters:**
- `data` — pointer to the byte array to checksum
- `len` — number of bytes (max 255 since it's `uint8_t`)

**Returns:** The 16-bit CRC value.

## How it is used in AC2 frames

The encoder (`AC2Framer::EncodeTelemetry`) calls:
```cpp
const std::uint16_t crc = CRC16(out_buf, pos);
out_buf[pos++] = static_cast<std::uint8_t>(crc & 0xFFU);  // CRC low byte
out_buf[pos++] = static_cast<std::uint8_t>(crc >> 8U);    // CRC high byte
```

CRC covers everything in the frame from SYNC to the last HMAC byte. It is appended as two bytes after the HMAC in little-endian order (low byte first).

The parser (`AC2Parser::Dispatch`) calls:
```cpp
const std::uint16_t expected_crc = CRC16(raw_, raw_len_);
const std::uint16_t received_crc = rx_crc_lo_ | (raw_[raw_len_] << 8U);
if (expected_crc != received_crc) { ++crc_errors_; Reset(); }
```

If the CRC doesn't match, the frame is silently dropped and the CRC error counter increments.

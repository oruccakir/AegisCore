## What this file is

`telemetry.hpp` is the **data dictionary** for the AC2 protocol. It defines:
- Command IDs (what type of message is this?)
- Error codes (why was a command rejected?)
- Fault codes (what kind of hardware fault occurred?)
- Payload structs (what bytes does each message carry?)

Every part of the system that constructs or interprets AC2 frames includes this header.

---

## `CmdId` namespace â€” Command IDs

Each AC2 frame has a 1-byte command ID field. These constants define all valid IDs:

```cpp
namespace CmdId {
    // Host â†’ Edge (commands from gateway to STM32)
    inline constexpr std::uint8_t kGetVersion    = 0x01U;  // ask for version info
    inline constexpr std::uint8_t kSetState       = 0x10U;  // change state machine state
    inline constexpr std::uint8_t kManualLock     = 0x11U;  // emergency stop / fail-safe
    inline constexpr std::uint8_t kResetAck       = 0x12U;  // acknowledge a reset (future)
    inline constexpr std::uint8_t kHeartbeat      = 0x99U;  // keep-alive from gateway

    // Edge â†’ Host (events/reports from STM32 to gateway)
    inline constexpr std::uint8_t kVersionReport  = 0x02U;  // version response
    inline constexpr std::uint8_t kReportState    = 0x20U;  // state changed
    inline constexpr std::uint8_t kTelemetryTick  = 0x21U;  // periodic health data
    inline constexpr std::uint8_t kFaultReport    = 0x30U;  // hardware fault occurred
    inline constexpr std::uint8_t kAuditEvent     = 0x31U;  // security/audit log entry

    // Bidirectional
    inline constexpr std::uint8_t kAck            = 0x80U;  // command accepted
    inline constexpr std::uint8_t kNack           = 0x81U;  // command rejected
}
```

The numbering groups related IDs:
- `0x01â€“0x1F`: commands from host
- `0x20â€“0x3F`: events from edge
- `0x80â€“0xFF`: protocol control (ACK/NACK)

## `ErrCode` namespace â€” NACK Error Codes

When a command is rejected (NACK), a 1-byte error code explains why:

```cpp
namespace ErrCode {
    inline constexpr std::uint8_t kInvalidCmd        = 0x01U;  // unknown command ID
    inline constexpr std::uint8_t kInvalidPayload    = 0x02U;  // payload too short/malformed
    inline constexpr std::uint8_t kInvalidTransition = 0x03U;  // state machine won't accept this
    inline constexpr std::uint8_t kAuthFail          = 0x04U;  // HMAC verification failed
    inline constexpr std::uint8_t kReplay            = 0x05U;  // sequence number is old
    inline constexpr std::uint8_t kRateLimited       = 0x06U;  // too many commands too fast
    inline constexpr std::uint8_t kFailSafeLock      = 0x07U;  // system is in fail-safe mode
    inline constexpr std::uint8_t kBusy              = 0x08U;  // cannot process right now
}
```

These are matched 1:1 with the `ERR_CODES` map in the React frontend (`useAC2Socket.ts`), which displays them as human-readable strings.

Sent in `kFaultReport` frames when the edge detects a hardware problem:

    inline constexpr std::uint8_t kPostFail      = 0x04U;  // POST check failed

## Payload structs

All structs are declared with `__attribute__((packed))`, which tells the compiler to not add padding bytes between fields. This ensures the byte layout of the struct exactly matches the protocol specification, regardless of the platform's alignment requirements.

Each struct has a `static_assert` that verifies its size at compile time â€” if the layout ever changes accidentally, the build fails.

### `PayloadReportState` (6 bytes)

```cpp
struct __attribute__((packed)) PayloadReportState
{
    std::uint8_t  state;        // current state (0=Idle, 1=Search, 2=Track, 3=FailSafe)
    std::uint8_t  prev_state;   // state we transitioned from
    std::uint32_t uptime_ms;    // milliseconds since boot at transition time
};
```

Sent whenever the state machine changes state (IDLEâ†’SEARCH, SEARCHâ†’TRACK, etc.). This is currently not yet wired up in `main.cpp` â€” state reports are only generated by the gateway from telemetry data.

### `PayloadTelemetryTick` (6 bytes)

```cpp
struct __attribute__((packed)) PayloadTelemetryTick
{
    std::uint8_t  state;                  // current state
    std::uint16_t cpu_load_x10;           // CPU % Ă— 10 (e.g., 153 = 15.3%)
    std::uint16_t free_stack_min_words;   // minimum free stack ever recorded
    std::uint8_t  hb_miss_count;          // heartbeat miss count
};
```

Sent every 1 second by `TelemetryTxTask`. The `cpu_load_x10` field is a placeholder (0) â€” actual CPU measurement requires `uxTaskGetSystemState()` with a results buffer.

### `PayloadFaultReport` (7 bytes)

```cpp
struct __attribute__((packed)) PayloadFaultReport
{
    std::uint8_t  fault_code;        // which type of fault (FaultCode::*)
    std::uint16_t ctx;               // additional context (e.g., reset reason bits)
    std::uint32_t reset_reason_bits; // RCC_CSR register captured after reset
};
```

Sent when a hardware fault is detected (currently only sent manually â€” automatic fault reporting on boot is a roadmap item).

### `PayloadHeartbeatTx` (4 bytes)

```cpp
struct __attribute__((packed)) PayloadHeartbeatTx
{
    std::uint32_t uptime_ms;   // milliseconds since boot
};
```

Sent every 1 second by `HeartbeatTask`. Lets the gateway know the edge node is alive and tells it the current uptime.

### `PayloadVersionReport` (15 bytes)

```cpp
struct __attribute__((packed)) PayloadVersionReport
{
    std::uint8_t  major;       // version major number
    std::uint8_t  minor;       // version minor number
    std::uint8_t  patch;       // version patch number
    std::uint8_t  git_sha[8];  // first 8 bytes of git commit SHA
    std::uint32_t build_ts;    // Unix timestamp of build
};
```

Sent in response to a `kGetVersion` command.

### `PayloadAck` (4 bytes)

```cpp
struct __attribute__((packed)) PayloadAck
{
    std::uint32_t echoed_seq;   // sequence number of the accepted command
};
```

### `PayloadNack` (5 bytes)

```cpp
struct __attribute__((packed)) PayloadNack
{
    std::uint32_t echoed_seq;   // sequence number of the rejected command
    std::uint8_t  err_code;     // reason for rejection (ErrCode::*)
};
```

## Why `inline constexpr` instead of `#define`?

`#define` is a text substitution â€” it has no type, no scope, no namespace. `inline constexpr` creates a real typed constant in the `CmdId::` namespace. This means:
- The compiler type-checks it
- It can be used in `switch` statements without warnings
- It lives in the namespace and won't collide with other code
